# Documentation Guide

## Purpose

This `docs/` directory contains repository-level documentation that supports the main learning path without duplicating the topic notes or lab instructions.

## Intended Content

- Study planning documents
- Reference guides that span multiple labs
- Repository conventions and documentation standards
- Supporting material that does not belong in a single lab or note

## Current Status

This folder is intentionally light during the initial scaffold phase. It will grow as the repository gains more validated content and cross-cutting documentation.

## Documentation Boundaries

- Use `labs/` for step-by-step practical exercises
- Use `notes/` for topic summaries and concept explanations
- Use `manifests/` for Kubernetes resource definitions
- Use `docs/` for higher-level repository documentation
