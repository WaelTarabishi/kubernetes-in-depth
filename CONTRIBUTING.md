# Contributing

Thank you for taking the time to contribute to `K8S-INDEPTH`.

This repository is primarily a Kubernetes learning and documentation project. Contributions should improve technical accuracy, clarity, structure, or reproducibility without overstating the maturity of the content.

## Contribution Principles

- Keep changes technically correct and easy to verify
- Prefer clear documentation over unnecessary complexity
- Do not add fake projects, fake metrics, or unvalidated production claims
- Keep examples aligned with the repository's learning-focused scope

## Ways to Contribute

- Fix documentation issues, typos, or broken links
- Improve explanations in notes or lab guides
- Suggest clearer verification steps
- Add references to official Kubernetes or CNCF documentation
- Improve repository organization while preserving existing learning content

## Pull Request Guidelines

1. Create a focused branch for your change.
2. Keep the scope small and reviewable.
3. Explain what changed and why.
4. Include validation notes when changing commands or technical instructions.
5. Avoid unrelated formatting churn.

## Documentation Style

- Use direct, professional language
- Prefer Markdown headings and short sections
- Use code blocks for commands and manifests
- Add placeholders only when content is intentionally deferred
- Link to primary documentation where possible

## Validation Expectations

If you add or modify technical instructions:

- Verify that commands are syntactically correct
- State prerequisites clearly
- Add a verification section when relevant
- Avoid presenting assumptions as established facts

## Issues

When opening an issue, include:

- The affected file or section
- A clear description of the problem
- Suggested correction, if available
- Supporting references for technical changes

## Code of Conduct

Be respectful, factual, and constructive in reviews and discussions.
